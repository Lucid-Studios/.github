# Engram Epistemic Model

## Purpose

Normalize Engram terminology across OAN Mortalis into explicit epistemic categories while preserving runtime behavior and backward compatibility.

## Canonical Knowledge Classes

1. Propositional Engrams
Declarative domain knowledge structures (facts, symbolic relations, structural laws).

2. Procedural Engrams
Executable reasoning pipelines represented as ordered functor chains.

3. Perspectival Engrams
Valuation and cognitive orientation structures used by compass-aligned interpretation.

4. Participatory Engrams
Role and interaction frameworks reserved for future subsystem implementation.

## Legacy Compatibility Mapping

- `Legendary Engram` -> `System Propositional Engram`
- `Constructor Engram` -> `Propositional Engram`
- `Master Engram` -> `Procedural Engram`

## Layered Cognition Interpretation

1. Propositional Engrams -> Knowledge
2. Procedural Engrams -> Reasoning
3. Perspectival Engrams -> Perspective
4. Participatory Engrams -> Participation

## Implementation Notes

- Existing runtime classes remain stable to preserve compatibility.
- Legacy labels are retained where needed for persisted payload continuity.
- New terminology is expressed in documentation, inline comments, and additive schema labels.
- No operational logic changes are introduced by this normalization patch.
