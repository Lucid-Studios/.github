# Lucid Studios

Lucid Studios maintains engineering repositories for symbolic systems, AI-adjacent runtime infrastructure, governance-oriented tooling, data-bearing artifact systems, and related research and implementation work.

## Organization Focus

- governed software and tooling
- trustworthy AI and artifact integrity
- symbolic and cognitive infrastructure
- security-conscious engineering workflows
- documentation and governance patterns that scale across repositories

## Standards Posture

Lucid Studios organizes repository work around:

- U.S.-aligned trustworthy AI and risk-management guidance
- provenance-aware data and artifact handling
- citizen science and public-contribution transparency where applicable
- GitHub community health, security, and contribution best practices

Shared organization guidance lives in the Lucid Studios `.github` repository.

## Working With Lucid Studios Repositories

Project issues, pull requests, workflows, and technical documentation should live in the relevant project repository unless the issue concerns shared organization governance or reusable workflows.

For project work, use the issue tracker and pull request flow in the target repository.

## Security

Do not post sensitive security details publicly. Use the repository security policy where available, or the Lucid Studios shared security guidance when the issue affects organization-level governance or reusable infrastructure.
