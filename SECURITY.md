# Security Policy

This repository provides Lucid Studios organization profile content, default community-health files, shared governance documents, and reusable workflows.

## Use the right reporting path

Do not open public issues here for:

- product-specific security defects
- credential exposure
- private key exposure
- repository-specific data leakage
- undisclosed vulnerabilities in project code

Use the affected repository's `SECURITY.md` when it exists. If the issue concerns shared Lucid Studios templates, workflows, or org-wide governance material, report it through GitHub security reporting for this repository if enabled, or contact maintainers privately.

## Shared-risk examples

Report here only when the issue affects organization-level materials such as:

- default contribution or security templates
- shared reusable workflows
- policy documents that create unsafe or misleading guidance
- org-wide standards that could weaken trust, provenance, or disclosure practices across repositories
