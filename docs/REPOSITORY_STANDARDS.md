# Lucid Studios Repository Standards

Lucid Studios uses this repository as the default source for shared GitHub health files, governance language, and reusable workflows.

## Core Principles

- preserve provenance and technical integrity
- prefer explicit risk disclosure over implied certainty
- use reviewable, auditable contribution workflows
- keep sensitive or restricted material out of public repositories
- use repository defaults as a baseline, with tighter product-local overrides when needed

## Minimum Shared Baseline

Repositories should have:

- a project `README.md`
- security guidance
- contribution guidance
- issue and pull request templates as appropriate
- explicit standards for AI use and data handling where the domain warrants it

## Override Model

Product repositories may override shared defaults when they need domain-specific guidance, but they should not weaken the shared Lucid Studios baseline casually.
